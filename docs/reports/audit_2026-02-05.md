## 🔴 Critical Issues (PHẢI SỬA NGAY)

### 1. ⚠️ HARDCODED SERVICE ROLE KEY

> [!CAUTION]
> **NGUY HIỂM NHẤT:** Đây là "chìa khóa chủ" của database. Ai có key này có thể xóa sạch toàn bộ dữ liệu!

**File:** [supabase_credentials.dart](file:///d:/okada/thu%20y/okada_vet_clinic/lib/core/constants/supabase_credentials.dart)

```dart
class SupabaseCredentials {
  static const String url = 'https://iasmrzpojtvxfsvskexc.supabase.co';
  static const String serviceRoleKey = 'eyJhbGci...'; // ❌ NGUY HIỂM!
}
```

**Hậu quả nếu không sửa:**
- Nếu app bị decompile (dễ với APK), hacker lấy được key này
- Key này bypass tất cả Row Level Security - có quyền làm BẤT CỨ GÌ với database
- Có thể xóa toàn bộ khách hàng, bệnh án, thuốc...

**Cách sửa:**
1. **NGAY LẬP TỨC:** Rotate key trong Supabase Dashboard (Settings → API → Regenerate Service Role Key)
2. Xóa file `supabase_credentials.dart`
3. Chỉ dùng `anon_key` trong client app (đã có trong `.env`)
4. Service role key chỉ dùng ở backend/server, KHÔNG BAO GIỜ trong client app

### 2. ⚠️ FILE .ENV CHỨA SERVICE ROLE KEY (ĐANG GHI NHẦM LÀ ANON)

**File:** [.env](file:///d:/okada/thu%20y/okada_vet_clinic/.env)

```
SUPABASE_ANON_KEY=eyJhbGci...InNlcnZpY2Vfcm9sZSI... 
                                    ↑ "service_role" = SAI!
```

**Vấn đề:** Comment nói "Anon/Public Key" nhưng thực tế đang dùng SERVICE ROLE key (nhìn payload JWT thấy `"role":"service_role"`).

**Cách sửa:**
1. Đăng nhập Supabase Dashboard → Settings → API
2. Copy đúng "anon public" key (KHÔNG PHẢI service_role)
3. Replace trong file `.env`
